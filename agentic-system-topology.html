AgenticSystemTopology 拓扑结构说明

这些枚举值描述的是一个 agentic system 的组织形态：有的是“节点类型”，有的是“多个节点如何协作”的结构。 下面每一块都给出直观图、中文含义和适合使用的场景。

AI_AGENT NON_AI_AGENT HUMAN_IN_THE_LOOP SEQUENCE PARALLEL LOOP ROUTER STAR

AI Agent AI_AGENT

一个由大模型驱动的智能节点。它可以理解任务、调用工具、生成回答或做决策。 在图里它通常是工作流中的一个“会思考的处理单元”。

像什么
客服助手、信息抽取器、代码审查 agent、规划 agent。
特点
输出不只是固定规则，可能依赖上下文、提示词和模型推理。

Non-AI Agent NON_AI_AGENT

一个不依赖大模型推理的确定性节点。它可以是普通函数、服务、数据库查询、规则引擎或 API 调用。

像什么
查库存、算价格、发邮件、读写数据库、格式化结果。
特点
逻辑更稳定，输入相同通常输出相同，适合承担可确定的业务动作。

Human In The Loop HUMAN_IN_THE_LOOP

流程中插入人的确认、补充或审批。系统可以先生成建议，但关键步骤需要人来判断。

像什么
审批退款、确认高风险操作、人工补全缺失信息。
特点
适合高风险、低容错或需要责任归属的流程。

Sequence SEQUENCE

多个节点按固定顺序执行，前一个节点的输出可以成为后一个节点的输入。 这是最常见的流水线结构。

像什么
抽取信息 → 查询数据 → 生成回复 → 输出结果。
特点
路径清晰，容易调试；缺点是后续步骤依赖前面步骤质量。

Parallel PARALLEL

多个节点同时处理同一个任务或同一份上下文，最后再汇总结果。 适合互不依赖的子任务。

像什么
同时让多个专家分析问题，再由汇总节点合成答案。
特点
速度快、覆盖面广；需要处理结果合并、冲突和失败分支。

Loop LOOP

某个节点或一组节点会反复执行，直到满足停止条件。常用于迭代优化、重试、反思和多轮处理。

像什么
生成草稿 → 检查质量 → 不合格则继续修改。
特点
能提升质量；必须设置终止条件，避免无限循环。

Router ROUTER

先由路由节点判断任务类型，再把请求分发给最合适的下游节点。 下游通常只会命中一个，或者命中一小组。

像什么
问题分类后转给售前、售后、技术支持或投诉处理。
特点
结构清楚，专业分工强；路由判断错了会影响后续结果。

Star STAR

一个中心节点协调多个外围节点。中心负责分配任务、收集结果、维护上下文或做最终决策。

像什么
主管 agent 调度多个专家 agent，最后统一输出。
特点
适合中心化协调；中心节点能力和稳定性会成为关键。
简单记忆：AI_AGENT、NON_AI_AGENT、HUMAN_IN_THE_LOOP 更像“节点类型”； SEQUENCE、PARALLEL、LOOP、ROUTER、STAR 更像“节点之间的组织结构”。
