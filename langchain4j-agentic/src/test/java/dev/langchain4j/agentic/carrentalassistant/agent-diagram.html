📋 图例说明

标记类型说明
Sequence 顺序编排 子Agent按声明顺序依次执行，上游输出作为下游上下文
Conditional 条件路由 根据 AgenticScope 中的状态动态选择激活哪些子Agent
Agent 单个Agent 独立的LLM调用单元，拥有自己的 chatModel 和 chatMemory
AgenticScope 共享状态 Agent间通过 writeState/readState 传递数据

🔄 执行流程

步骤Agent数据产出
①CustomerInfoExtractionService提取客户信息 → state[customerInfo]
②TowingAgentService处理拖车请求 → state[towingResponse]
③-1EmergencyExtractorService识别紧急事件 → state[emergencies]
③-2EmergencyExperts (beforeCall)解析emergencies → state[fire/medical/policeEmergency]
③-2aFireAgentService火灾建议 → state[fireResponse] (条件触发)
③-2bMedicalAgentService医疗建议 → state[medicalResponse] (条件触发)
③-2cPoliceAgentService报警建议 → state[policeResponse] (条件触发)
③-3EmergencyResponseService汇总紧急响应 → state[emergencyResponse]
④ResponseGeneratorService最终输出 → state[response]
