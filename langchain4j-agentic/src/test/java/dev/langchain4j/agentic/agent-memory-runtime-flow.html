初始化 SequentialPlanner + 进入 PlannerLoop
PlannerLoop.loop()
▶
Planner planner = plannerSupplier.get(); // new SequentialPlanner() planner.init(new InitPlanningContext(currentScope, this, subagents)); // SequentialPlanner.init() 记住 subagents = [Assistant代理] Object result = new PlannerLoop(planner, currentScope, registry).loop();
// PlannerLoop.loop() 核心循环 nextAction = planner.firstAction(new PlanningContext(agenticScope, null)); while (nextAction == null || !nextAction.isDone()) { List<AgentExecutor> agents = ((Action.AgentCallAction) nextAction).agentsToCall(); switch (agents.size()) { case 1 -> agents.get(0).execute(agenticScope, this); // ← 串行执行 } }
PlannerLoop 是调度引擎：它不断问 Planner "下一步该做什么？"，然后执行。 Planner 只返回 Action（调用谁 or 结束），PlannerLoop 负责真正执行并回调。
6
SequentialPlanner.firstAction() → 返回 Assistant
SequentialPlanner
▶
// firstAction() 默认调用 nextAction() public Action nextAction(PlanningContext ctx) { return terminated() ? done() : call(agents.get(agentCursor++)); // agents.size()=1, cursor=0 → 返回 call(Assistant代理) // cursor 自增为 1 } public boolean terminated() { return agentCursor >= agents.size(); // 1 >= 1 → true }
因为只有1个子 Agent，第一次 nextAction() 返回 call(Assistant)，cursor 变为1。 下一轮循环时 terminated()=true，会返回 done()，循环结束。

AgentInvocationHandler：从 ThreadLocal 取 Scope，转发到 AiServices
AgentInvocationHandler.invoke()
▶
public Object invoke(Object proxy, Method method, Object[] args) { // method = processData(String), args = ["构建一个机器人"] AgenticScope scope = LangChain4jManaged.current(AgenticScope.class); // ← 从 ThreadLocal 取出，非 null！说明在 Agentic 系统内部 if (scope != null) { return method.invoke(agent, args); // ↑ agent 是 aiServices.build() 的产物（第3层代理） // 反射调用会进入 DefaultAiServices 的 InvocationHandler } return invokeStandaloneAgent(method, args); // 独立调用时才走这 }
关键判断：LangChain4jManaged.current(AgenticScope.class) 用来区分两种调用模式：
• 嵌入模式（scope ≠ null）：作为 Agentic 系统子节点运行，Scope 由父级提供
• 独立模式（scope = null）：单独调用，自行创建临时 Scope

核心机制总结

三层代理嵌套

Sequence代理(PlannerBased) → AI Agent代理(AgentInvocation) → LLM代理(DefaultAiServices)。每层都是 JDK Proxy，method.invoke() 层层穿透。

Scope = 共享黑板

DefaultAgenticScope 是 ConcurrentHashMap 包装的状态容器。子 Agent 写 outputKey → 父 Agent 读 outputKey，实现跨层通信。

PlannerLoop 调度模式

Planner 只返回 Action（调用谁/结束），PlannerLoop 执行并回调 onSubagentInvoked 驱动下一步。解耦了"决策"与"执行"。

ChatMemory 自动管理

DefaultAiServices 在调用 LLM 前自动从 ChatMemory 读历史，调用后自动追加 UserMessage + AiMessage。MessageWindowChatMemory 保留最近10条。

ThreadLocal 传递 Scope

AgentInvoker 把 AgenticScope 放入 LangChain4jManaged ThreadLocal，AgentInvocationHandler 从中取出。避免显式传参。

outputKey 自动持久化

AgentExecutor 在子 Agent 执行后自动 writeState(outputKey, response)。父 Agent 通过 readState(outputKey) 获取最终结果。
