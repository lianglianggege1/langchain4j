PlannerLoop 调度循环
PlannerLoop
▶
public Object loop() { nextAction = planner.firstAction(new PlanningContext(agenticScope, null)); while (nextAction == null || !nextAction.isDone()) { List<AgentExecutor> agents = ((Action.AgentCallAction) nextAction).agentsToCall(); nextAction = null; switch (agents.size()) { case 1 -> agents.get(0).execute(agenticScope, this); // 串行 default -> parallelExecution(agents); // 并行 } } return result(); // 从 scope 读 outputKey }
SequentialPlanner 每次 nextAction() 返回一个子 Agent， PlannerLoop 负责真正调用它，并在完成后通过 onSubagentInvoked() 驱动 Planner 继续下一步。

核心机制总结

三层代理嵌套

Workflow代理(PlannerBased) → AI Agent代理(AgentInvocation) → LLM代理(DefaultAiServices)。每层都通过 JDK Proxy 实现，method.invoke() 层层穿透。

Scope 状态共享

所有 Agent 共享同一个 DefaultAgenticScope（内部是 ConcurrentHashMap）。子 Agent 通过 writeState 写入结果，后续 Agent 通过 readState 读取。

PlannerLoop 调度

Planner 只决定"下一步调用谁"（返回 Action），PlannerLoop 负责真正执行。Action 完成后通过 onSubagentInvoked 回调驱动 Planner 继续。

outputKey 自动持久化

AgentExecutor 在子 Agent 执行完毕后，自动把 response 写入 AgenticScope[outputKey]。父 Agent 通过相同的 key 读取结果。
